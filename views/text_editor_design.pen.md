### 1.2 颜色方案

- **背景色**: #0f0f23 (深色主背景)
- **面板背景**: #1e1e2e (深色面板背景)
- **边框色**: #333 (分隔线)
- **文本色**: #e0e0e0 (主文本)
- **次要文本**: #888 (标签、提示)
- **强调色**: #667eea (主要按钮、链接)
- **成功色**: #4ade80 (发送按钮)
- **错误色**: #f87171 (错误提示)

### 1.3 字体方案

- **主要字体**: 系统默认无衬线字体
- **代码字体**: Monaco, Consolas, 'Courier New'
- **字体大小**: 13-16px (正文), 12-14px (标签), 14-18px (标题)
- **行高**: 1.5-1.6 (正文), 1.2 (标题)

## 五、核心交互流程

### 5.1 AI 辅助写作流程

1. **创建模块**: 用户点击右侧「新建」按钮，中间工作区创建空白文本模块
2. **输入提示词**: 用户在右侧输入框中输入提示词，如 "帮我写一封会议邀请邮件"
3. **选择模型**: 用户选择一个或多个AI模型
4. **发送请求**: 点击发送按钮或使用快捷键发送请求
5. **接收结果**: AI流式返回结果，实时显示在左侧「当前文本」和中间模块中
6. **优化内容**: 用户继续输入指令优化内容，如 "把语气改得更正式一些"
7. **版本管理**: 系统自动保存历史版本，用户可以展开左侧历史面板查看和对比版本

### 5.2 多模型对比流程

1. **输入提示词**: 用户输入提示词
2. **选择多个模型**: 勾选多个模型 (如 GPT-4o 和 Claude)
3. **发送请求**: 点击发送按钮，并行请求多个模型
4. **查看结果**: 界面显示多个模型的结果，并排对比
5. **选择版本**: 用户选择最终版本，应用到当前文本

## 八、技术实现要点

### 8.1 组件结构

```
src/views/
└── AiTextEditorView.vue          # 主视图

src/components/text-editor/
├── LeftPanel.vue                 # 左侧面板
│   ├── ModelSelector.vue         # 模型选择
│   ├── SyncSelector.vue          # 同步选择
│   ├── CurrentText.vue           # 当前文本
│   └── HistoryPanel.vue          # 历史版本
├── WorkArea.vue                  # 中间工作区
│   └── TextModule.vue            # 文本模块
├── RightPanel.vue                # 右侧面板
│   ├── ChatHistory.vue           # 对话历史
│   ├── ModelCheckbox.vue         # 模型多选
│   └── ChatInput.vue             # 输入框
├── CompareView.vue               # 版本对比视图
└── AiToolbar.vue                 # AI 工具栏
```

### 8.2 状态管理

- 使用 Vue 3 Composition API + provide/inject
- 管理模块状态、对话历史、版本历史
- 实现跨组件通信

### 8.3 性能优化

- 虚拟滚动处理长对话历史
- 节流处理文本编辑事件
- 批量更新DOM操作
- 懒加载非关键组件

### 8.4 可访问性

- 支持键盘导航
- 提供屏幕阅读器支持
- 合理的颜色对比度
- 清晰的视觉反馈
